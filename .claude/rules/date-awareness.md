# 日付認識ルール

> 日付・年号の誤出力を防止するルール

---

## 原則

**日付を出力する際は、必ず現在の日付を確認してから出力する。**

現在の日付は Claude Code の環境変数 `Today's date` に含まれている。
SessionStart hook でも `CURRENT_DATE` として注入される。

---

## 禁止事項

- 2024年、2025年など過去の年号をデフォルトで使用しない
- 「来週」「先週」などの相対日付を使う場合、必ず絶対日付も併記する
- 日付が不明な場合は推測せず「要確認」と記載する

---

## チェックリスト

日付を含む出力を行う前に確認:

1. 現在の年は正しいか（環境変数を参照）
2. 曜日・日付の計算は正しいか
3. 相対日付（来週、先月など）は絶対日付に変換したか

---

## 出力例

```
✅ 正しい例:
   期限: 2026-02-01（金）
   作成日: 2026-01-28

❌ 間違い例:
   期限: 2024-02-01  ← 年が古い
   作成日: 来週      ← 絶対日付がない
```

---

## 適用範囲

- TASKS.yaml のタイムスタンプ
- DECISIONS.md の日付
- RUN_LOG.md のエントリ
- OWNER_INBOX.md の質問
- 全ての台帳ファイル
