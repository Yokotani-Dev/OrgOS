# LEARNINGS

> プロジェクト固有の学習記録

---

## このディレクトリについて

`/org-learn` コマンドで抽出された「再利用可能なパターン」を保存する場所。

セッション中に解決した問題から、将来役立つ知見を蓄積する。

---

## 保存される内容

| カテゴリ | 説明 |
|----------|------|
| エラー修正 | 根本原因と解決策（類似問題に適用可能） |
| デバッグ手法 | 効果的だったツールやアプローチ |
| ワークアラウンド | ライブラリの癖、API制約、バージョン固有の回避策 |
| プロジェクト規約 | このプロジェクト固有のパターンやアーキテクチャ |

---

## ファイル命名規則

```
YYYY-MM-DD-<slug>.md
```

例:
- `2026-01-21-supabase-rls-gotcha.md`
- `2026-01-22-nextjs-caching-issue.md`

---

## ファイルフォーマット

```markdown
# <タイトル>

## 問題

<何が起きたか>

## 原因

<なぜ起きたか>

## 解決策

<どう解決したか>

## 適用条件

<いつこの知識を使うべきか>

## 参照

- <関連ドキュメントやIssue>
```

---

## 使い方

### 学習の記録

```
/org-learn
```

セッション中の作業を振り返り、再利用可能なパターンを抽出して保存。

### 活用

- 新規メンバーのオンボーディング資料として
- 類似問題発生時の参照として
- プロジェクトのベストプラクティス集として

---

## 注意事項

- 些細な修正（typo、単純なバグ）は記録しない
- 将来の時間節約につながるパターンのみ記録
- 機密情報（API キー、パスワード）は含めない
