## Acceptance Criteria

### A1: pre-exec-validate.sh
`bash scripts/codex/pre-exec-validate.sh <TASK_ID>`
- 引数: TASK_ID
- 動作:
  1. `.ai/TASKS.yaml` から TASK_ID の autonomy_level / allowed_paths / risk_level を抽出
  2. autonomy_level=owner_only なら **block** (Codex 委任禁止、Owner 直接のみ)
  3. allowed_paths が KERNEL_FILES (T-OS-371 で拡張済み) と重複していたら block
  4. allowed_paths が空 or 過剰広範 (例: `**`) なら block
  5. validation pass なら exit 0、fail なら exit 非 0 + 明確なエラー

### A2: post-exec-audit.sh
`bash scripts/codex/post-exec-audit.sh <TASK_ID> <worktree_path>`
- 引数: TASK_ID, worktree_path (Codex が走った worktree)
- 動作:
  1. `git -C <worktree_path> diff --name-only HEAD` で変更ファイル一覧取得
  2. 各 file が allowed_paths 範囲内か check
  3. 範囲外なら **revert** (`git -C <worktree_path> checkout HEAD -- <file>`) + log
  4. KERNEL_FILES に該当する変更があれば **必ず revert** + critical log
  5. audit 結果を `.ai/CODEX/AUDIT/<TASK_ID>.yaml` に記録

### A3: check-autonomy-runtime.sh
`bash scripts/authority/check-autonomy-runtime.sh <TASK_ID>`
- 引数: TASK_ID
- 動作:
  1. autonomy_level + risk_level + reversibility から authority-layer.md の Decision Matrix で expected outcome を判定
  2. risk_level=critical かつ autonomy_level != owner_only → 矛盾、block
  3. high + irreversible なのに autonomy_level=silent_execute → 矛盾、block
  4. matrix 通過なら exit 0

### A4: 検証
- 各 script を `bash -n` で syntax check
- pre-exec で owner_only task が block される (test fixture)
- post-exec で allowed_paths 外への変更が revert される (test fixture)
- check-autonomy-runtime で矛盾組合せが block される
- shellcheck 通過
