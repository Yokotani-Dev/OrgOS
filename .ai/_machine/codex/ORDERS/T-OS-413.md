### 2. `scripts/org/request-integration.sh` (新規)

Manager が queue に pending item を atomic に作成。

#### 引数
```bash
request-integration.sh \
  --task-id T-XXX \
  --worktree-path <path> \
  --branch task/T-XXX-slug \
  --base-branch main \
  --artifact-manifest <path> \
  --commit-message "<msg>" \
  [--diff-patch <path>] \
  [--handoff <path>] \
  [--priority 50] \
  [--verifier-status passed|skipped]
```

#### 動作
1. 入力検証:
   - `--task-id` 必須、`T-OS-XXX` 形式
   - artifact_manifest 存在確認 + JSON parse
   - worktree_path 存在確認
   - branch が `task/<task_id>-...` 形式
   - protected branch (main/develop) なら拒否
2. queue item id 生成: `IQ-$(date -u +%Y%m%dT%H%M%SZ)-<task_id>-$(uuid 8hex)`
3. queue item JSON 構築 (schema v1)
4. `.ai/queue/integration/pending/<task_id>.json` への atomic write:
   - 一時 file `.ai/queue/integration/pending/<task_id>.json.tmp` に書き
   - `mv` で rename (atomic)
5. exit 0、queue item path を stdout

duplicate task_id pending item があれば拒否 (exit 3)。

### 3. `scripts/org/integrator-commit.sh` (新規)

commit 権限を持つ唯一の経路。queue item を consume して commit する。

#### 引数
```bash
integrator-commit.sh \
  --task-id T-XXX \
  [--queue-item PATH]
```

#### 動作 (queue-driven)
1. `.claude/state/git.lock` を flock で取得 (existing logic 再利用 or 簡易版)
2. queue item を読む:
   - `--queue-item` 指定 or `.ai/queue/integration/pending/<task_id>.json` default
3. queue item 検証:
   - schema parse 成功
   - status == "pending"
   - artifact_manifest が存在 + `verify-artifact-manifest.py` で valid
   - worktree path 存在
   - branch が `task/<task_id>-...` 形式 + protected branch ではない
   - diff が allowed_paths 内 (`git -C <worktree> diff --name-only base_commit..HEAD`)
   - diff budget 内 (max_files, max_lines)
   - dependencies (tasks) が全 done
   - verifier status == passed または required==false
4. queue item を `pending/` → `processing/` に atomic move
5. commit 実行:
   - `cd <worktree>`
   - `git -C <worktree> commit -m <message> --author=...` 
   - **bypass mechanism**: `ORGOS_INTEGRATOR=1` 環境変数を export してから git commit を呼ぶ。pretool_policy.py はこの env を見て integrator commit を許可する。
6. commit SHA 取得、queue item に `commit.commit_sha` を記録
7. queue item を `processing/` → `done/<YYYYMM>/<task_id>.<integrated_at>.json` に move
8. exit 0

失敗時:
- queue item を `processing/` → `failed/` に move
- `attempts.count += 1`、`last_attempt_at`, `last_error` を記録
- `attempts.count < max` なら pending に戻す option (本 task では実装せず failed のまま)
- exit 非ゼロ
