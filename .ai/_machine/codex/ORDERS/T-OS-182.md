## Acceptance Criteria

### A1: proactive-mode.md 新規ルール
```markdown
# Proactive Mode — OrgOS 真髄の核

## Iron Law
Owner が起動 or 明示的に「何からやる？」と聞いた際、Manager は以下を実行:

1. Session Bootstrap を完了
2. suggest-next.sh で次アクション候補を生成 (3-5 個)
3. 各候補に推奨度 (P0/P1/P2) と理由を付記
4. Owner が即実行できる形式で提示

## 起動トリガー
- "何からやる" / "次は" / "どう進める" のキーワード
- "朝です" / "起きました" のような挨拶 (朝一発言)
- session-bootstrap の直後で `awaiting_owner=false` かつ `queued tasks > 0` の場合

## 提案ロジック
1. GOALS.yaml の active milestones を読む
2. TASKS.yaml の queued + deps 解消済みタスクを抽出
3. 各タスクに priority score 計算:
   - score = priority_weight (P0=10, P1=5, P2=1)
   - + blocker_release_bonus (deps を解除するタスクに +2)
   - + recent_momentum (直近完了タスクと関連 +1)
   - + owner_preference_match (USER_PROFILE から +0〜2)
4. 上位 3-5 件を提案

## Red Flags
- 提案に理由なし → Owner が判断できない
- Owner preference 無視 → 症状 A (気が利かない)
- 進行中タスクと無関係な提案 → context_miss
```
