## Background

機密情報 (API キー、トークン、パスワード) の保存ガイドラインと helper script。
1Password CLI / macOS Keychain / SOPS のいずれかを採用。`.claude/rules/secret-management.md` で policy 定義 + `scripts/org/secret-get.sh` で取得 helper。

## Choice

**macOS Keychain** を採用 (Owner 環境が macOS、追加 install 不要、簡単)。

## Required

### `.claude/rules/secret-management.md` (新規)
- secret は `.ai/`, `.claude/`, `outputs/`, `scripts/` に直書き禁止
- pretool で secret pattern を検出して deny する旨を記載 (実装は別 task)
- secret 取得は `scripts/org/secret-get.sh` 経由のみ
- Keychain entry の登録手順、削除手順

### `scripts/org/secret-get.sh` (新規)
```bash
# Usage: scripts/org/secret-get.sh <service> <account>
# Returns: secret to stdout
# Uses: security find-generic-password -s <service> -a <account> -w
```

### `scripts/org/secret-set.sh` (新規)
```bash
# Usage: scripts/org/secret-set.sh <service> <account> [--read-stdin | --prompt]
# Stores secret in Keychain (security add-generic-password)
```

### `tests/kernel/test-secret-management.sh` (新規)
- mock keychain で get/set 動作確認 (or skip if real keychain unavailable)
- secret-get.sh が exit non-zero on missing entry
