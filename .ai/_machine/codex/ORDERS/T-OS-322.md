## Context

`.claude/rules/authority-layer.md` は `silent_execute / execute_with_report / ask_before_execute / owner_only` の 4 段の autonomy levels を定義しているが、実際の `.ai/TASKS.yaml` の各 task には autonomy metadata が **0 件 backfill されていない**。結果として Manager は「これは自動でよい」を判断する材料がなく、安全側に倒して Owner 確認に逃げている (Codex Phase 2 RC-3)。

本タスクで全 active / queued task に **autonomy metadata** を機械的に backfill する。**判定ルールは決定論的** で、LLM 判断に依存させない。

- `.ai/REVIEW/T-OS-300/SYNTHESIS.md` §3
- `.ai/REVIEW/T-OS-300/CODEX_ANALYSIS_v2.md` §2.5 + RC-3
- `.claude/rules/authority-layer.md` (read-only 参照)

## Acceptance Criteria

### A1: Schema 定義
`.claude/schemas/autonomy.yaml` を新規作成 (or 既存ファイルなら拡張):

```yaml
$schema: orgos/autonomy/v1
fields:
  autonomy_level:
    type: enum
    values: [silent_execute, execute_with_report, ask_before_execute, owner_only]
    required: true
  blast_radius:
    type: enum
    values: [single_file, multi_file, schema_change, iron_law, kernel]
    required: true
  owner_input_needed:
    type: array_of_enum
    values: [vision, budget, compliance, credentials, destructive_approval, none]
    required: true
  risk_level:
    type: enum
    values: [low, medium, high, critical]
    required: true
  default_if_no_response:
    type: enum
    values: [auto_apply, defer, escalate, no_op]
    required: false
  reversibility:
    type: enum
    values: [reversible, irreversible]
    required: true
```

### A2: 決定論的判定ルール
タスクの `allowed_paths` と `owner_role` から autonomy 値を機械決定するルールを implement:

```
Rule 1: allowed_paths に CLAUDE.md / AGENTS.md / manager.md / authority-layer.md / request-intake-loop.md / memory-lifecycle.md / handoff-protocol.md を含む
  → autonomy_level: owner_only
  → blast_radius: kernel
  → risk_level: critical
  → reversibility: reversible
  → owner_input_needed: [destructive_approval]

Rule 2: allowed_paths に既存 .claude/rules/ の他ファイルを含む
  → autonomy_level: ask_before_execute
  → blast_radius: schema_change
  → risk_level: high
  → reversibility: reversible
  → owner_input_needed: [destructive_approval]

Rule 3: allowed_paths に既存 .claude/schemas/ または .claude/agents/ の他ファイルを含む
  → autonomy_level: ask_before_execute
  → blast_radius: schema_change
  → risk_level: medium
  → reversibility: reversible
  → owner_input_needed: [none]

Rule 4: allowed_paths が src/ や 既存実装ファイル (拡張子 .ts/.py/.js/.rb/.go) のみ
  → autonomy_level: execute_with_report
  → blast_radius: multi_file
  → risk_level: medium
  → reversibility: reversible
  → owner_input_needed: [none]

Rule 5: allowed_paths が docs/ / .ai/RESOURCES/ / 新規ディレクトリのみ
  → autonomy_level: silent_execute
  → blast_radius: single_file
  → risk_level: low
  → reversibility: reversible
  → owner_input_needed: [none]

Rule 6: 上記いずれにも当てはまらない (.ai/ 配下台帳更新等)
  → autonomy_level: execute_with_report
  → blast_radius: single_file
  → risk_level: low
  → reversibility: reversible
  → owner_input_needed: [none]
```

判定後、`default_if_no_response` は autonomy_level から導出:
- silent_execute / execute_with_report → auto_apply
- ask_before_execute → defer
- owner_only → escalate

### A3: Backfill 実装
TASKS.yaml の全 `status: queued` または `status: running` または `status: blocked` または `status: review` の task に対して、上記ルールで autonomy fields を追加。

**既存フィールド (id / title / status / deps / priority / owner_role / allowed_paths / acceptance / notes 等) は一切変更しない**。autonomy fields のみ append。

`status: done` や `status: archived` の task には backfill しない (既に完了したため不要)。

### A4: Coverage Check Script
`scripts/authority/check-autonomy-coverage.sh` を新規作成:
- TASKS.yaml の active / queued / blocked / review tasks のうち autonomy fields が欠けている task を列挙
- exit 1 if any missing
- `--json` フラグで JSON 出力

### A5: 検証
- `bash scripts/authority/check-autonomy-coverage.sh` で 0 件 (全部 backfill 済み) を確認
- TASKS.yaml の YAML としての valid 性を確認 (既存 parser を壊さない)
- diff を確認し、autonomy fields 以外が変更されていないことを確認
