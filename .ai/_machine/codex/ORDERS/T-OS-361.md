## Acceptance Criteria

### A1: セッション開始時のブランチ記録
- pretool_policy.py に `record_session_branch()` 関数追加
- セッション開始 (もしくは hook 初回呼び出し) で `git branch --show-current` を実行
- 結果を `.claude/state/expected_branch_<sessionId>` に記録 (timestamp 付き)
- sessionId は環境変数 or hook input から取得

### A2: git 操作前のチェック
- pretool_policy.py の `verify_pretool` で Bash(git ...) を検出
- `git checkout` `git switch` `git commit` `git merge` `git rebase` `git stash` を対象
- 現在ブランチ (`git branch --show-current`) と expected_branch を比較
- 不一致なら `block(...)` で実行を止め、Owner に明示メッセージを返す

### A3: bypass option
- `expected_branch` がない場合 (新セッション初回) は record してから許可
- Owner が明示的に branch 切替を意図する場合のみ `--update-expected` フラグを持つ helper を提供
  - `bash scripts/git/update-expected-branch.sh <new_branch>`

### A4: helper script
`scripts/git/check-branch-consistency.sh`:
- 引数なし: 現在の expected vs current を表示、不一致なら exit 1
- `--report`: JSON で結果を返す (CI / monitor 用)

### A5: 検証
- pretool_policy.py の existing test (あれば) を破壊しない
- 新規テスト: branch mismatch 検出シナリオ
- shellcheck と python lint を通す
