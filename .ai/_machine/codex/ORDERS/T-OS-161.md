### A4: coherence-mode.md 新規ルール
`.claude/rules/coherence-mode.md`:

```markdown
# Coherence Mode — deterministic 判定 rubric

## Iron Law
応答の Coherence mode は以下の rubric で deterministic に決定する。主観判定禁止。

## 3 段階

### Silent Bind (文脈を応答に出さない)
条件 (以下の全てを満たす):
- 依頼が単純な読み取り/フォーマット操作 (grep, format, list)
- 依頼が 10 単語以下
- TASKS.yaml の running タスクとの依存関係なし
- 応答が 3 行以下で完結

例:
- "このログから error を抽出して"
- "JSON を整形して"

### Brief Bind (1 行だけ位置づけを書く)
条件 (以下のいずれかを満たす):
- 依頼が進行中タスク (TASKS.yaml status=running) と関連
- 依頼が最近の DECISIONS (過去 7 日) に関連
- 応答が 3-10 行

例:
- "このファイルにログ出力追加して" (→ 進行中の debug タスクに関連)

### Full Bind (背景+影響+選択肢+推奨を明示)
条件 (以下のいずれかを満たす):
- 依頼が新規方針/仕様に関わる
- Owner の判断が必要 (request-intake-loop Step 6 で ask 判定)
- 複数タスクへの影響がある
- リスク分類が medium 以上

例:
- "認証方式を JWT に変える"
- "デプロイ環境を AWS に移す"

## Deterministic 判定ロジック

```python
def coherence_mode(request, active_tasks, recent_decisions, risk_level):
    word_count = len(request.split())
    affects_tasks = any(task.keywords in request for task in active_tasks)
    affects_decisions = any(d.keywords in request for d in recent_decisions)
    
    if risk_level in ["medium", "high", "critical"]:
        return "Full"
    if word_count > 30 or affects_decisions:
        return "Full"
    if affects_tasks or (word_count > 10):
        return "Brief"
    return "Silent"
```

## 違反検出
- 全応答に mode 判定ログを trace に残す (handoff-packet.verification.self_check で記録)
- Manager Quality Eval の context_miss_rate で検出
```
