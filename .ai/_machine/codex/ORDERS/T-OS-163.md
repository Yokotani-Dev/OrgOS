## Acceptance Criteria

### A1: Regression Detection 自動化
- 最新 jsonl と直前の jsonl を比較
- 前回 pass → 今回 fail のケースを列挙
- 前回 target 達成 → 今回 未達の指標を列挙
- 退行時は exit code 2 (fail とは別のシグナル)
- payload を `.ai/METRICS/manager-quality/regression-YYYY-MM-DD.md` に出力
- DECISIONS.md への追記は Manager が判断 (本タスクでは payload 生成のみ)

### A2: decision_trace_completeness rubric 具体化
metrics.yaml の decision_trace_completeness の formula を具体化:

```yaml
decision_trace_completeness:
  formula: |
    A decision is "fully traced" if ALL of:
      - has assumptions >= 1 (or explicit "no assumptions" acknowledgement)
      - has decisions_made >= 1 (rationale + alternatives_considered filled)
      - has verification (tests_run or eval_results non-empty)
      - has memory_updates (or explicit "no updates" acknowledgement)
      - trace.span_id, trace.request_trace_id present
    full_trace_count / total_decisions
  target: "> 95%"
  priority: P0
  weight: 1.5
```

report.py の判定ロジックに Handoff Packet の検査を組み込む。
(Handoff Packet が未実装のタスクは "legacy" として除外可)

### A3: owner_delegation_burden trend 計算
- 直近 7 日分の jsonl から、Owner に依頼した件数 / 全タスク数 を算出
- 7 日移動平均を計算
- downward trend の判定: 直近 3 日 MA < 直近 7 日 MA (少なくとも 3 日分のデータがある場合のみ)
- データ不足時は "pending" ステータスで pass 扱い
- scripts/eval/trend-calculator.sh として実装、report.py から呼び出し

### A4: regression-detection.sh の独立 script
