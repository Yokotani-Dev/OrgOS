## Context

OrgOS Manager は業界知識 (法令/業界 policy/競合制約) を持たない。Owner も全業界の専門家ではない。
そのため regulated domain (広告/医療/金融/不動産など) のプロジェクトで、業界制約を知らないまま実装が進み、
あとから「景表法に違反していた」「Meta HOUSING カテゴリのポリシー違反だった」が発覚するパターンが構造的に発生する。

これを防ぐには **BRIEF / REQUIREMENTS フェーズで「この project の domain constraint」を明示化する protocol** が必要:
- 何の業界か
- どの法令/規制が関係するか
- 業界 policy (プラットフォーム規約など) が関係するか
- Owner と擦り合わせて confirmed にする
- Web 検索や専門家入力を経て補完する

これは Quality Contract (T-OS-350) と Journey-First (T-OS-351) の隣に位置する **Owner Touchpoint Type A** の追加 protocol。

## Acceptance Criteria

### A1: Schema 作成
`.claude/schemas/domain-constraint.yaml` を新規作成。以下を必須フィールドとする:
- id (`DC-XXX-YYY` パターン)
- related_milestone, related_tasks
- domain_category (enum: advertising, medical, financial, real_estate, education, gaming, dating, crypto, b2b_general, b2c_general, internal_tool, other)
- regulations (array: 法令/規則の id + name + summary + source_url + last_verified_at)
- platform_policies (array: e.g., Meta HOUSING, Apple App Store, Google Play, Stripe TOS)
- prohibited_practices (array: 明示的にやってはいけないことのリスト)
- required_practices (array: 必ず実装すべきことのリスト)
- knowledge_gaps (array: Manager/Owner が未確認の領域、後続調査タスク必要)
- sync_status (draft | confirmed | superseded)
- confirmed_at, confirmed_by, expires_at (法令は変わるので有効期限必須)
- notes

### A2: Iron Law Rule 作成
`.claude/rules/domain-constraint-sync.md` を新規作成。以下を含む:

- Iron Law section: 5 項目以上 (regulated domain では DOMAIN_ANALYSIS confirmed が DESIGN gate の前提、Manager 独断禁止、knowledge_gaps 解消なしの実装禁止 など)
- Domain Analysis Workshop プロセス (BRIEF 直後の 5 step: domain identification → regulation discovery → platform policy check → prohibited/required mapping → Owner confirm)
- regulated domain 判定ルール (どの project で本 rule が起動するか)
- DESIGN Gate 接続 (DOMAIN_ANALYSIS confirmed が無いと DESIGN 進行禁止)
- Knowledge Gap Handling (Owner も知らない領域は WebSearch / 専門家相談タスクに変換)
- Red Flags (10 項目程度)
- Violation Detection / Response
- Relationship To Other Rules (quality-contract.md, user-journey-sync.md, request-intake-loop.md)

### A3: Template 作成
`.ai/TEMPLATES/DOMAIN_ANALYSIS.md` を新規作成。実プロジェクトでコピーして使う Markdown テンプレート:
- Project / Milestone refs
- Domain Category 選択
- Regulations table
- Platform Policies table
- Prohibited Practices リスト
- Required Practices リスト
- Knowledge Gaps リスト (調査タスクへの参照)
- Sync Status block
- Owner Confirmation block
