## Background

Plan Contract schema (T-OS-454) is defined. Now we need a generator that takes a Codex Work Order (`.ai/CODEX/ORDERS/T-OS-XXX.md`) and produces `.plan.yaml` automatically, extracting `title`, `acceptance`, and `allowed_paths`.

## Required

### `scripts/org/generate-plan.py` (新規)

```python
# Usage: python3 scripts/org/generate-plan.py T-OS-XXX
# Reads: .ai/CODEX/ORDERS/T-OS-XXX.md
# Writes: .ai/plans/T-OS-XXX.plan.yaml
# Parses markdown sections:
#   - title from H1 (after task id)
#   - acceptance from "## Acceptance" bullets
#   - allowed_paths from "## Allowed Paths" bullets
#   - verification: defaults to ["bash tests/kernel/run-kernel-tests.sh"]
```

Validation:
- Loaded `.plan.yaml` must pass `.claude/schemas/plan-contract.v1.json`
- Exit non-zero on parse failure or missing sections
- Output is yaml.safe_dump with sort_keys=False

### `tests/kernel/test-generate-plan.sh` (新規)

- Missing "## Allowed Paths" section fails
- Atomic write (temp file → rename)
- Idempotent (re-running produces same output)
