## Changed Scripts

- `scripts/authority/request-approval.sh`
  - Creates `.ai/APPROVALS/<UUID>.yaml` with `pending` status.
  - Appends an approval request block to `.ai/OWNER_INBOX.md`.
  - Emits the request UUID to stdout.
  - Writes request lifecycle audit lines to `.ai/AUDIT/approval-YYYY-MM-DD.log`.
- `scripts/authority/check-approval.sh`
  - Reads `.ai/APPROVALS/<UUID>.yaml`.
  - Scans `.ai/OWNER_COMMENTS.md` for `approve <UUID>` or `reject <UUID>`.
  - Updates request status to `approved`, `rejected`, `expired`, `applied`, or `applied_and_failed`.
  - Emits JSON status to stdout.
- `scripts/authority/approval-workflow.sh`
  - Wraps request + optional polling.
  - Non-wait mode returns UUID immediately.
  - `--wait` polls every 30 seconds by default for up to 24 hours.
- `scripts/authority/apply.sh`
  - When `os-mutation-check.sh` returns `next_step: request_approval`, calls `approval-workflow.sh --wait`.
  - Applies the patch only after `approved`.
  - Skips on `rejected`, `pending` timeout, or `expired`.
  - Marks approval as `applied_and_failed` if dry-run or apply fails after approval.

## apply.sh Integration

`apply.sh` now accepts optional metadata:

```bash
scripts/authority/apply.sh \
  --task-id T-OS-XXX \
  --operation edit_manager_md \
  --target .claude/agents/manager.md \
  --patch /path/to/change.patch \
  --summary "リファクタリング" \
  --impact "Tick フロー変更"
```

Flow:

1. `os-mutation-check.sh` evaluates the operation.
2. If `next_step` is `request_approval`, `approval-workflow.sh --wait` creates an Owner request and waits.
3. `approved` continues to backup, dry-run, and `git apply`.
4. `rejected`, `expired`, or timeout returns `status: skipped`.
5. Post-approval dry-run/apply failures mark the approval as `applied_and_failed`.

## Owner Usage Notes

OWNER_INBOX entries use this shape:

```markdown

- Operation: edit_manager_md
- Target: .claude/agents/manager.md
- Summary: リファクタリング
- Impact: Tick フロー変更
```

Owner response format:

- Approve: `echo "approve <UUID>" >> .ai/OWNER_COMMENTS.md`
- Reject: `echo "reject <UUID>" >> .ai/OWNER_COMMENTS.md`
