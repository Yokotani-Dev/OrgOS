# Runbooks

プロジェクトのルーティンワークを手順化・再利用可能にする仕組み。

## Runbookとは

同じ種類の作業（デプロイ、リリース、マイグレーション等）を毎回ゼロから考えるのではなく、
手順を一度記録し、以降はその手順に従って機械的に実行するためのドキュメント。

## いつ作るか

- 同じ種類の作業を2回以上実行したら、Manager が自動で Runbook を作成する
- プロジェクト開始時に明らかなルーティン（デプロイ、リリース）は初回実行後すぐに作成
- Owner が「これ手順化して」と指示した場合

## 構成

各 Runbook は以下の要素を含む:
- **概要**: この手順が何をするか
- **前提条件**: 事前に必要な状態・ツール
- **手順**: ステップバイステップの実行手順
- **検証**: 成功の確認方法
- **ロールバック**: 問題発生時の戻し手順
- **履歴**: 変更の記録

## 管理ルール

1. **Runbook がある作業は必ず Runbook に従う** — 自己流は禁止
2. **手順に問題があったら Runbook を更新** — 場当たり的な回避は禁止
3. **実行結果を履歴に記録** — 成功/失敗/改善点
4. **TEMPLATE.md をベースに作成** — 統一フォーマット
