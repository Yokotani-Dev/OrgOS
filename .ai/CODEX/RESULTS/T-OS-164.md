## 3. 各 case の修正内容

### MQ-009
- `scenario` を map 化し、`summary` / `active_tasks` / `stable_tasks` を追加。
- 現行 judge が `scenario` を参照しないため、`input.active_tasks` は使わず、Decision ベースで文脈 bind を判定できる形へ調整。
- `input.decisions` には `.ai/DECISIONS.md` の設計原則「検証可能な文脈・能力・権限・記憶に基づいて」を設定。

### MQ-010
- `scenario` を map 化し、並列タスクの `active_tasks` / `stable_tasks` を追加。
- 現行 judge に合わせて `input.decisions` を使用し、`.ai/DECISIONS.md` の並列実行設計文言に bind。
- expected / anti-pattern も「並列実行の衝突確認」に寄せて更新。

### MQ-017
- 存在しない `scripts/release/check-preview.sh` を削除。
- 実在し、再生成後に `CAPABILITIES.yaml` へ登録される `script_memory_check_no_plain_secrets` を参照するよう変更。
- `current_request` も script の責務に合わせて更新。

### MQ-018
- 実環境に存在しない `filesystem` MCP を削除。
- strict judge に合わせて dict 形式の capability 指定へ変更。
- `cli_git` と `cli_jq` の既知手段を再利用するケースに一般化。

### scan.sh / CAPABILITIES
- `scan_scripts()` を `scripts_root.rglob("*.sh")` に変更し、`scripts/` 配下の shell script を再帰検出する形に明示。
- 再生成により `scripts/memory/check-no-plain-secrets.sh` を含む script capability 群が `.ai/CAPABILITIES.yaml` に登録された。
