# Handoff Packet: T-OS-373

## Summary
- `scripts/git/check-branch-consistency.sh`
  - Added `--block-if-mismatch`.
  - Added `--require-lock`.
  - Existing `--report` and `--update-expected` behavior is preserved.
- `.claude/hooks/pretool_policy.py`
  - Detects git mutation commands: `checkout`, `switch`, `commit`, `merge`, `rebase`, `stash`, `push`.
  - Starts `scripts/git/acquire-lock.sh` before mutation gates.
  - Requires branch consistency through `check-branch-consistency.sh --block-if-mismatch --require-lock`.
  - Releases a newly acquired policy lock when the policy later blocks the command.
- `scripts/codex/run-in-worktree.sh`
  - Calls `pre-exec-validate.sh <TASK_ID>` before worktree creation.
  - Calls `post-exec-audit.sh <TASK_ID> <worktree_path>` after Codex exits.
  - Logs pre/post output as structured `key=value` events.

## Rationale
- Branch mismatch blocking is now available as a direct script mode and is used by the hook gate.
- Git mutation operations now require a same-session `.claude/state/git.lock` holder before proceeding.
- Codex worker launches now pass through the T-OS-372 boundary validation and post-exec audit layer.

## Risk & Rollback
- Risk: `pretool_policy.py` now leaves a successful git mutation lock held for the actual tool execution. Operators must release it with `scripts/git/release-lock.sh` after the protected mutation flow.
- Risk: worktree wrapper now aborts earlier if Work Order metadata fails pre-exec validation.
- Rollback: revert the three changed files listed below.
