## Implementation Notes

- `run-in-worktree.sh` accepts `bash scripts/codex/run-in-worktree.sh <TASK_ID> [--keep-worktree]`.
- It creates `.worktrees/<TASK_ID>` from `HEAD`, runs Codex from inside that worktree, and removes the worktree unless `--keep-worktree` is set.
- TASK_ID rejects empty values, path separators, leading dots, trailing hyphen, and non `[A-Za-z0-9._-]` characters.
- Worktree creation failure logs a structured error, runs `git worktree prune`, attempts to remove an empty partial directory, and exits `1`.
- Missing TASK_ID and invalid options exit `2`.
- Cleanup uses `git worktree remove --force` so disposable Codex edits do not block teardown.
