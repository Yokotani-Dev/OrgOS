## 3. 10 ステップの要約

1. Intake: 依頼原文を固定し、追跡可能な起点にする
2. Load Relevant Memory: `USER_PROFILE` と `past_qa` から再利用可能な記憶を回収する
3. Bind Active Work Graph: `CONTROL` / `TASKS` / `GOALS` に依頼を結び付ける
4. Discover Capabilities: `CAPABILITIES.yaml` から自動実行可能な手段を先に探す
5. Classify Risk / Reversibility: 可逆性、コスト、セキュリティ、破壊度を分類する
6. Decide: matrix と inquiry budget に従って `act` / `ask` / `defer` / `refuse` を決める
7. Execute with Trace: 実行または質問に trace と推奨を付ける
8. Verify: 自己報告ではなく期待値照合と副作用確認を行う
9. Update TASKS / DECISIONS / MEMORY: 状態変化と再利用価値を台帳へ反映する
10. Report with Minimal Cognitive Load: Silent / Brief / Full Bind で認知負荷を最小化して報告する
