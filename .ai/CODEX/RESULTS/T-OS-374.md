## Rationale
- The breaker stops apply -> event -> detect -> apply recursion by enforcing hard static limits before apply preflight.
- Counter mutation is explicit and structured, so every breaker state transition emits JSON logs and successful application records preserve the post-apply counter snapshot.
- Iron Law protection is additive: existing forbidden path/prefix checks remain, while KERNEL_FILES and current Iron Law rule files are now unconditional automatic-apply rejects.

## Risk and Rollback
- Risk: `increment-apply` trips the breaker when the configured max is reached, so the third successful apply in the default cycle leaves the breaker open and the fourth apply is rejected.
- Risk: scheduler timeout fallback uses Python on systems without GNU `timeout`; this keeps macOS smoke checks runnable but depends on Python 3.
- Rollback: revert the changed scripts and remove `.ai/EVOLUTION/circuit-breaker.yaml`; for state-only recovery, Owner can run `bash scripts/evolution/circuit-breaker.sh restore`.
