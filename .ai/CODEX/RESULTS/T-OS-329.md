## Rationale

The scheduler is a wrapper around the existing evolution pipeline rather than a
new engine. Default scheduled operation stays in `shadow`, so automatic runs
record decisions and traces without modifying target files. The dry-run mode
keeps validation artifacts in `.ai/SCHEDULER` to avoid mutating `.ai/EVOLUTION`
during smoke checks.

## Risk And Rollback

Risk is limited to scheduler entry points and runtime state. Production runs can
append to `.ai/EVOLUTION/events.jsonl`, write proposal files, and write shadow
application records, matching the existing evolution script contract.

Rollback:

- Remove or disable `.github/workflows/orgos-scheduler.yml`.
- Do not install the launchd template or cron entry.
- Remove `scripts/scheduler/` and `.ai/SCHEDULER/` if the scheduler is not used.
