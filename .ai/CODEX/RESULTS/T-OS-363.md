## Implementation Notes

- `bash scripts/git/acquire-lock.sh [--timeout 30]` acquires `.claude/state/git.lock` and holds it until process termination.
- The implementation uses Python `fcntl.flock` from a bash wrapper because this environment does not provide a `flock` command.
- On success, it writes structured holder metadata: `pid`, `sessionId`, `acquired_at`, `lock_file`, and `worktree_path`.
- On timeout, it exits `1` and prints the current recorded holder pid, session id, and acquisition time.
- `bash scripts/git/release-lock.sh` reads the holder pid, sends `SIGTERM`, and verifies the kernel lock becomes available.
- Stale metadata is reported for manual review; no automatic stale-lock cleanup was added.
