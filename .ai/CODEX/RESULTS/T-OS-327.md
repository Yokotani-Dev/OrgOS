# T-OS-327 Result + Handoff Packet

## Summary

Implemented offline-first adaptive capability probing and requirement-based role routing.

- Added JSON Schema 2020-12 capability role schema at `.claude/schemas/capability-roles.yaml`.
- Added `scripts/capabilities/probe-new.sh` to compare `.ai/CAPABILITIES.example.yaml` against stub/fixture sources and emit T-OS-324 style evolution proposals.
- Added `scripts/capabilities/role-routing.sh` to recommend semver-style model aliases by `role_id`.
- Added `scripts/capabilities/README.md` with usage, fixture shape, and out-of-scope notes.

## Rationale

The implementation keeps T-OS-327 inside the Quality Contract boundary:

- No real provider API calls are made.
- Model recommendations use aliases such as `reasoning-primary@^1.0.0`, not provider model identifiers.
- Probe output is proposal-only and does not automatically switch models or mutate routing.
- Missing source fixtures are logged as graceful skips.
- Routing includes a semantic equivalence placeholder for manual follow-up.

## Risk & Rollback

Primary risks:

- `.claude/schemas/capability-roles.yaml` is both schema and default routing catalog via `x-orgos-default-instance`.
- `probe-new.sh` defaults to writing proposals under `.ai/EVOLUTION/proposals/`; smoke verification used a temp output directory to avoid repo proposal churn.

Rollback:

- Remove the files listed above.
- No shared ledger files were modified.
