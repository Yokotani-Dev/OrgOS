## Background

Codex CLI を sandbox 内で起動すると、`--output-last-message ../../.ai/CODEX/RESULTS/T-OS-XXX.txt` への書き込みが macOS sandbox により `operation not permitted` で拒否される事象が T-OS-412/413/414 で 3 回発生。

Codex は fallback で `/private/tmp/OrgOS-<task>-handoff.txt` に書いている。Manager は wrapper streaming output から拾えているが、これは durable interface ではない。

5th round GPT-5.5 Q28 #4 / Q24 #5:
> Codex に main repo `.ai/CODEX/RESULTS` へ直接書かせようとする設計をやめる。wrapper が worktree 内 / stdout / tmp から artifact store に吸い上げる。canonical source は `.ai/artifacts/<task>/<run>/output-last-message.txt` にする。

## 修正方針

**Codex の `--output-last-message` を worktree 内 path に向ける**。main repo への書き込みを試みない (sandbox 制約と整合)。wrapper は完了後 artifact store にコピー。canonical source は `.ai/artifacts/<task>/<run>/output-last-message.txt`。

## Required Implementation

### 1. `scripts/codex/run-in-worktree.sh` 修正

`--output-last-message` の引数を **worktree 内** path に変更。

```bash
# 旧 (おそらく):
codex_cmd=(/opt/homebrew/bin/codex exec --full-auto --skip-git-repo-check \
  --output-last-message "../../.ai/CODEX/RESULTS/$task_id.txt" - )

# 新:
worktree_handoff_path="$worktree_path/.orgos-handoff/output-last-message.txt"
mkdir -p "$(dirname "$worktree_handoff_path")"
codex_cmd=(/opt/homebrew/bin/codex exec --full-auto --skip-git-repo-check \
  --output-last-message "$worktree_handoff_path" - )
```

`worktree内/.orgos-handoff/` は post-exec-audit の allowed_paths にデフォルトで含まれない可能性があるが、wrapper が事前に作るので worktree-internal でしか動かない (audit の対象は worktree 内ファイルの変更だが、`.orgos-handoff/` は wrapper 生成なので revert 対象から除外する logic が必要)。

実装 option:
- (a) `.orgos-handoff/` を post-exec-audit が無視する
- (b) `--output-last-message` を `/tmp/orgos-<task_id>-<run_id>-handoff.txt` に向け、wrapper が artifact store にコピー
- **推奨 (b)**: tmp 経由のほうが audit を汚さない、shell が `/tmp` を sandbox 許容しているのも実証済み

### 2. `scripts/codex/run-in-worktree.sh` artifact 吸い上げ

## Acceptance Criteria

1. wrapper が Codex `--output-last-message` を tmp 経由 (e.g., `/tmp/orgos-<task>-<run>-handoff.txt`) に向ける
2. wrapper が完了後その tmp file を artifact store にコピー (`.ai/artifacts/<task>/<run>/output-last-message.txt`)
3. main repo の `.ai/CODEX/RESULTS/<task>.txt` への書き込みを試みない (sandbox issue 回避)
4. Codex 完了 / 失敗 どちらでも tmp file → artifact store コピーが best-effort 実行 (失敗時は warning、cleanup を block する)
5. test 1 件以上 (codex handoff capture via tmp)
6. run-kernel-tests.sh で既存 + 新規 全 test pass
7. `git add` / `git commit` / `git push` 一切実行しない
