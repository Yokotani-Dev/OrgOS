## Background

T-OS-418 dogfood retry (T-OS-422+423 fix 後) で別 path leak:

```
changed path is outside allowed_paths: .ai/ARTIFACTS/T-OS-412/.../artifact_manifest.json
```

macOS は case-insensitive FS。`.ai/artifacts/T-OS-412/` と `.ai/ARTIFACTS/T-OS-412/` は同じディレクトリだが、git は最初に記録した case (ARTIFACTS uppercase) を維持。INTERNAL_PATHS は Python の str.startswith で case-sensitive なので `.ai/ARTIFACTS/` をマッチしない。

## Fix

```python
# 旧
if any(path.startswith(prefix) for prefix in INTERNAL_PATHS):
    continue

# 新
if any(path.lower().startswith(prefix.lower()) for prefix in INTERNAL_PATHS):
    continue
```

prohibited / allowed_paths check も同様に case-insensitive 化が安全 (macOS で誤 deny を防ぐ)。

ただし allowed_paths は user-supplied なので慎重に: glob match (fnmatch) は元々 macOS で case-insensitive 寄り だが Python の fnmatch は case-sensitive 動作。

スコープ判断: 本 task では **INTERNAL_PATHS のみ case-insensitive** にする (filter は permissive にしてもセキュリティ低下しない、allowed_paths check は user 意図通り case-sensitive のまま保つほうが予測可能)。
