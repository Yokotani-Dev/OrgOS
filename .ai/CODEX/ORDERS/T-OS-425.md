## 設計再考

integrator-commit.sh の責務 = **lease.allowed_paths 内の user diff のみを安全に commit**。
git status の他の path は **integrator にとって irrelevant** (Codex / runtime / OS / 過去残骸)。

### 旧 algorithm (whack-a-mole)
```
1. git status --porcelain → 全 changed paths
2. INTERNAL_PATHS startswith filter → 残った paths
3. 残った paths を allowed_paths check
4. fail なら refuse、pass なら git add + commit
```

問題: step 2 の filter が「漏れ」を生む。新しい runtime-managed path が追加されるたび iteration。

### 新 algorithm (intersect approach)
```
1. git status --porcelain → all_changed_paths
2. all_changed_paths ∩ allowed_paths (path_covers match) → user_diff_paths
3. user_diff_paths が空 → "no user diff to commit" として fail (空 commit 防止)
4. user_diff_paths を prohibited check → 該当あれば refuse
5. user_diff_paths のみ git add + commit
6. その他の changed paths は無視 (integrator 責務外)
```

利点:
- INTERNAL_PATHS list 不要 (撤廃)
- 新しい runtime path 追加で壊れない
- 「allowed_paths にあるもののみ commit」という意図が直接コード化

### 1. `scripts/org/integrator-commit.sh` 改訂

L270-291 付近の changed_files 収集 + L293-323 の allowed_paths check を統合し、intersect approach に書き換え。

```python
# 概念コード
import fnmatch
import subprocess
import sys
from pathlib import Path

worktree = sys.argv[1]
queue_path = Path(sys.argv[2])

with queue_path.open() as f:
    data = json.load(f)
scope = data.get("scope") or {}
allowed = [str(p) for p in (scope.get("allowed_paths") or [])]
prohibited = [str(p) for p in (scope.get("prohibited_paths") or [])]
if not allowed:
    print("scope.allowed_paths must not be empty"); sys.exit(1)

def matches(path: str, pattern: str) -> bool:
    # Path-prefix match (for dir patterns) + fnmatch (for globs) + exact
    if path == pattern:
        return True
    if pattern.endswith("/") and path.startswith(pattern):
        return True
    if fnmatch.fnmatch(path, pattern):
        return True
    return False

# 1. all changed paths
output = subprocess.check_output(["git", "-C", worktree, "status", "--porcelain"], text=True)
all_paths = []
for line in output.splitlines():
    if len(line) < 4:
        continue
    path = line[3:]
    if " -> " in path:
        path = path.split(" -> ", 1)[1]
    all_paths.append(path)
all_paths = sorted(set(all_paths))

# 2. intersect with allowed_paths
user_diff = [p for p in all_paths if any(matches(p, a) for a in allowed)]

# 3. empty check
if not user_diff:
    print("no user diff within allowed_paths"); sys.exit(1)

# 4. prohibited check (against user_diff only)
for p in user_diff:
    if any(matches(p, q) for q in prohibited):
        print(f"changed path is prohibited: {p}")
        sys.exit(1)

# Output user_diff for shell side
for p in user_diff:
    print(p)
```

## Note: 「security 後退」の懸念

deny-list (INTERNAL_PATHS) は「想定外の変更があったら refuse」という defensive な意図があった。新 allow-list は「想定外を無視」する。

しかし integrator の責務は **user intent を commit する** こと。想定外 path の detection / quarantine は **post-exec-audit.sh の責務** (これは worker 側の audit、既に動いている)。役割分離として allow-list が clean。

万一「想定外 detection」を integrator 側でも望むなら、commit 後に warning log を出す:
```
warned_unused_paths = all_paths - user_diff - {known runtime paths}
if warned_unused_paths:
    print(f"WARN: status had unexpected paths (not committed): {warned_unused_paths}")
```

これは optional。本 task では実装不要。
