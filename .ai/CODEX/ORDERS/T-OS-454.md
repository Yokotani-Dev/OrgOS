## Background

Week 7 introduces the Plan Contract (`.plan.yaml`) — a per-task spec that Manager writes BEFORE dispatching Codex, defining acceptance criteria, allowed_paths, and verification steps. The integrator gate (T-OS-457) will require `.plan.yaml` to exist and validate before allowing commit.

## Required

### `.claude/schemas/plan-contract.v1.json` (新規)

JSON Schema for `.plan.yaml`:

```yaml
schema_version: orgos.plan_contract.v1
task_id: T-OS-XXX
title: "..."
acceptance:
  - "Test X passes"
  - "Y file created"
allowed_paths:
  - scripts/foo.sh
  - tests/kernel/
verification:
  - command: "bash tests/kernel/test-X.sh"
    expect: pass
created_at: "ISO8601"
created_by: "manager|owner"
```

Required fields:
- `schema_version` (const: "orgos.plan_contract.v1")
- `task_id` (pattern: `T-OS-\d+`)
- `title` (non-empty)
- `acceptance` (array, min 1)
- `allowed_paths` (array, min 1)
- `verification` (array, min 1)
- `created_at` (ISO 8601)
- `created_by` (enum: manager, owner, codex)
