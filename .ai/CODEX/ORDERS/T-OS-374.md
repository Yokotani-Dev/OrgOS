## Acceptance Criteria

### A1: circuit-breaker.yaml schema
```yaml
version: "1.0"
limits:
  max_apply_per_cycle: 3
  max_apply_per_day: 10
  consecutive_revert_threshold: 3
  scheduler_timeout_minutes: 30
state:
  current_cycle_apply_count: 0
  today_apply_count: 0
  consecutive_revert_count: 0
  last_apply_at: null
  breaker_state: "closed"  # closed / half_open / open
  tripped_at: null
  trip_reason: null
```

### A2: circuit-breaker.sh
- `bash scripts/evolution/circuit-breaker.sh check` — exit 0 if closed, exit 1 if open
- `bash scripts/evolution/circuit-breaker.sh increment-apply` — counter ++
- `bash scripts/evolution/circuit-breaker.sh increment-revert` — counter ++、threshold 到達で trip
- `bash scripts/evolution/circuit-breaker.sh reset-cycle` — cycle counter reset
- `bash scripts/evolution/circuit-breaker.sh reset-daily` — daily counter reset (cron 用)
- `bash scripts/evolution/circuit-breaker.sh trip <reason>` — manual trip
- `bash scripts/evolution/circuit-breaker.sh restore` — Owner 用 (reset)

### A3: apply.sh 強化
- 各 apply 前に `circuit-breaker.sh check` を call、open なら exit
- apply 成功 → `circuit-breaker.sh increment-apply`
- rollback 実行時 → `circuit-breaker.sh increment-revert`
- 既存の forbidden file check を確認 + 強化:
  - KERNEL_FILES (T-OS-371 で拡張) への apply は無条件 reject
  - rationalization-prevention.md / authority-layer.md / Iron Law 全 file
- iteration counter を application-record に記録

### A4: scheduler timeout
`scripts/scheduler/run-detection.sh` に:
- `timeout 1800 bash scripts/evolution/detect.sh` (30 min default)
- 各 sub-step の timeout
- timeout 発生時は circuit-breaker trip

### A5: 検証
- circuit-breaker check / increment / trip / restore が動く
- max_apply_per_cycle=3 到達で apply.sh が拒否
- consecutive_revert=3 で trip
- KERNEL_FILES (T-OS-371) への apply が reject
- shellcheck 通過
