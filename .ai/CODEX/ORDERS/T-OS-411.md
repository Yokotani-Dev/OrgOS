### 1. `.claude/schemas/artifact-manifest.v1.json` (新規)

JSON Schema 完全版を作成。仕様は `.ai/REVIEW/T-OS-400/external-ai-4th-response.md` の Q17 セクションにある「JSON Schema」をそのままコピー (約 200 行)。

主要構造:
- `schema_version: "orgos.artifact_manifest.v1"` const
- top-level: `project_id`, `task_id`, `run_id`, `created_at`, `repo`, `actor`, `execution`, `artifacts[]`, `limits`, `verification`, `cleanup`
- `repo`: `repo_root`, `worktree_path`, `branch`, `head_before`, `head_after`, `dirty_after`

- `limits`: `max_log_bytes`, `max_file_bytes`, `max_total_bytes`

- `cleanup`: `cleanup_allowed`, `cleanup_reason`, `cleanup_status`

### 2. `scripts/org/collect-artifacts.sh` (新規)

artifact 収集の本体スクリプト。

#### 引数
```bash
collect-artifacts.sh \
  --task-id T-OS-XXX \
  --run-id <run_id> \
  --worktree-path <path> \
  --artifact-dir .ai/artifacts/<task_id>/<run_id> \
  --stdout-source <path> \
  --stderr-source <path> \
  --last-message-source <path> \
  --actor-role codex|manager|subagent|integrator|mock \
  --actor-id <id>
```

#### 実行内容
1. artifact_dir 作成 (`logs/`, `files/untracked/`, `files/generated/`, `audit/` サブディレクトリ)
2. stdout/stderr を copy
3. `output-last-message` を copy
4. `git -C <worktree> status --porcelain=v1 > artifact_dir/git-status.txt`
5. `git -C <worktree> diff --binary > artifact_dir/git-diff.patch`
6. `git -C <worktree> diff --cached --binary > artifact_dir/git-diff-cached.patch`
7. untracked files を `git ls-files --others --exclude-standard -z` で列挙し copy (symlink は target を `<file>.symlink-target` に保存、followed=false)
8. 各 artifact の sha256 と size を計算
9. `artifact_manifest.json` を生成 (Python helper を呼ぶ)
10. `artifact_manifest.sha256` を生成

#### limits
- stdout/stderr: 20MB まで保存。超過時は先頭 10MB + 末尾 10MB に切り詰めて `kind: truncated_log`
- single file: 50MB まで。超過時は `kind: skipped_large_file` で metadata のみ
- total artifact dir: 200MB まで
- `node_modules`, `.git`, build cache は保存対象外

### 3. `scripts/org/verify-artifact-manifest.py` (新規)

Day 0 の `verify_artifact_manifest_minimal()` (shell 内 Python heredoc) を完全版に置き換える。

#### 動作
```python
# verify-artifact-manifest.py <manifest_path>

# 1. JSON parse
# 2. required top-level fields 確認
# 3. 各 artifact_path が manifest directory 配下の relative path であることを確認
#    - absolute path 禁止
#    - ".." 禁止
# 4. 各 artifact:
#    - status == captured かつ required == true なら存在必須
#    - size_bytes と実 file size 一致
#    - sha256 一致
# 5. required artifact が missing/skipped/truncated なら fail
# 6. verification.verified が true であることを確認
# 7. fail → exit 1, pass → exit 0
```

エラー時は stderr に詳細を出す。

### 4. `scripts/codex/run-in-worktree.sh` の更新

#### 4-1. run_id 生成
```bash
run_ts="$(date -u +%Y%m%dT%H%M%SZ)"
rand="$(uuidgen | tr 'A-F' 'a-f' | cut -c1-8)"
run_id="${run_ts}-${task_id}-${rand}"
```

正規表現: `^[0-9]{8}T[0-9]{6}Z-[A-Za-z0-9._-]+-[a-f0-9]{8}$`

#### 4-2. stdout/stderr capture
```bash
artifact_dir="$repo_root/.ai/artifacts/$task_id/$run_id"
mkdir -p "$artifact_dir/logs"

stdout_file="$artifact_dir/logs/stdout.log"
stderr_file="$artifact_dir/logs/stderr.log"
last_msg_file="$artifact_dir/output-last-message.txt"

set +e
"${codex_cmd[@]}" \
  > >(tee -a "$stdout_file") \
  2> >(tee -a "$stderr_file" >&2)
codex_exit=$?
set -e
```

`script` command は使わない (Linux/macOS 挙動差が大きい)。

#### 4-3. Codex 実行後の自動 artifact collection
codex 実行後 (cleanup 前) に `collect-artifacts.sh` を呼ぶ。

#### 4-4. verify_artifact_manifest_minimal() を置き換え
`verify-artifact-manifest.py` を呼ぶ形に。Day 0 の minimal 関数は削除。

#### 4-5. cleanup 経路の更新
Day 0 で実装した cleanup logic は維持。違いは `artifact_manifest_path` が自動的に `$artifact_dir/artifact_manifest.json` を指すようになる (引数で override 可能)。
