### 1. `scripts/org/validate-tasks-yaml.py` (新規)

duplicate key を error にする YAML loader を使い、`.ai/TASKS.yaml` を validate する。

```python
#!/usr/bin/env python3
"""Validate .ai/TASKS.yaml against duplicate keys and schema."""
import sys
from pathlib import Path

try:
    from ruamel.yaml import YAML, constructor
    USE_RUAMEL = True
except ImportError:
    USE_RUAMEL = False

def validate(path):
    if USE_RUAMEL:
        # ruamel.yaml は duplicate key を DuplicateKeyError として raise
        yaml = YAML(typ="safe")
        yaml.allow_duplicate_keys = False
        with open(path) as f:
            try:
                data = yaml.load(f)
            except constructor.DuplicateKeyError as e:
                print(f"DUPLICATE_KEY: {e}", file=sys.stderr)
                return 1
    else:
        # fallback: PyYAML で load 後、line scan で duplicate-id を検出
        import yaml
        with open(path) as f:
            content = f.read()
            data = yaml.safe_load(content)
        # task id duplication check
        ids = [t['id'] for t in data.get('tasks', []) if isinstance(t, dict) and 'id' in t]
        seen = {}
        for i, tid in enumerate(ids):
            seen.setdefault(tid, []).append(i)
        dupes = {k: v for k, v in seen.items() if len(v) > 1}
        if dupes:
            print(f"DUPLICATE_TASK_ID: {dupes}", file=sys.stderr)
            return 1
        
        # task entry boundary check (heuristic): consecutive same-key fields in one entry
        # Look for patterns like:
        #   priority: P0
        #   ...
        #   priority: P1   <- second occurrence within same entry = orphan
        # Walk YAML tokens and detect.
        import re
        current_task = None
        seen_keys_in_task = {}
        for ln, line in enumerate(content.splitlines(), 1):
            m_id = re.match(r'^\s*-\s+id:\s+', line)
            if m_id:
                current_task = ln
                seen_keys_in_task = {}
                continue
            m_key = re.match(r'^(\s{4,})(\w+):\s', line)
            if m_key and current_task:
                key = m_key.group(2)
                if key in seen_keys_in_task and key not in ('-',):
                    # duplicate key within same task entry
                    prev_ln = seen_keys_in_task[key]
                    print(f"DUPLICATE_KEY_IN_TASK: line {ln} duplicates '{key}' from line {prev_ln} (task started at line {current_task})", file=sys.stderr)
                    return 1
                seen_keys_in_task[key] = ln
        return 0
    
    # also basic schema check (id, title, status required)
    for t in data.get('tasks', []):
        if not isinstance(t, dict):
            continue
        for required in ('id', 'title', 'status'):
            if required not in t:
                print(f"MISSING_FIELD: {required} in task {t.get('id', '?')}", file=sys.stderr)
                return 1
    return 0

if __name__ == "__main__":
    path = sys.argv[1] if len(sys.argv) > 1 else ".ai/TASKS.yaml"
    sys.exit(validate(path))
```

ruamel.yaml が利用可能なら使う、そうでなければ heuristic で十分。

### 2. `scripts/org/update-task.py` (新規)

`.ai/TASKS.yaml` の単一 task entry の field を安全に更新する CLI。direct edit の代替。

```bash
# Usage examples
scripts/org/update-task.py T-OS-416 --set status=done
scripts/org/update-task.py T-OS-416 --set 'allowed_paths=[".claude/hooks/pretool_policy.py", "tests/kernel/"]'
scripts/org/update-task.py T-OS-416 --add-note "実装完了 2026-05-16"
scripts/org/update-task.py --list-ids
scripts/org/update-task.py T-NEW-001 --create --title "..." --status queued --priority P1
```

主な動作:
1. ruamel.yaml で round-trip (コメント保持、format 保持) で load
2. 指定 task entry を find
3. field 更新
4. 書き出し前に validate-tasks-yaml.py を内部で呼んで pass しないと書かない
5. atomic write (tmp file + mv)

`--create` で新規 task entry を作成、boundary を正確に管理 (直接 Edit の代替)。
