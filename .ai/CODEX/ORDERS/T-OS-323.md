## Context

Phase 2 SYNTHESIS の中核概念: OS が「何を持っているか」を自己記述する DNA を持つ。
- `.ai/ORG_DNA.yaml`: rules / agents / skills / capabilities / schemas / commands を versioned に表現
- generated / managed / owner-edited の 3 区分タグ
- `.ai/DNA_HISTORY.yaml`: semver-style version diff を記録
- `scripts/dna/regenerate.sh`: DNA から `.orgos-manifest.yaml` view を再生成
- 将来の Marketplace / AI 進化追従の基盤

## Acceptance Criteria

### A1: Schema (.claude/schemas/org-dna.yaml)
JSON Schema 2020-12 準拠。最低以下のフィールド:
- version (semver: e.g., "0.1.0")
- generated_at, generator (manager / codex / owner)
- components: rules / agents / skills / capabilities / schemas / commands
  - 各 component: { id, path, version, classification (generated/managed/owner-edited), checksum, depends_on }
- metadata: project_name, vision_ref, primary_milestones

### A2: 初期 DNA 生成 (.ai/ORG_DNA.yaml)
- 既存 `.claude/rules/`, `.claude/agents/`, `.claude/skills/`, `.claude/schemas/`, `.ai/CAPABILITIES.example.yaml`, `.claude/commands/` を walk
- 各ファイルを component として登録
- 既存 `.orgos-manifest.yaml` の情報を参考に classification を付与 (managed が default、明示既存 owner edit なら owner-edited)
- version: "0.1.0"

### A3: Regenerator (scripts/dna/regenerate.sh)
- 引数なし: ORG_DNA.yaml を読み、現状の `.claude/` と `.ai/CAPABILITIES.example.yaml` を re-scan、DNA を update
- `--dry-run`: 差分のみ表示
- `--bump-version <patch|minor|major>`: semver bump
- update 時は DNA_HISTORY.yaml に previous version の snapshot を append

### A4: Diff Script (scripts/dna/diff.sh)
- `scripts/dna/diff.sh <version_a> <version_b>`: DNA_HISTORY から 2 version の diff を表示
- output: added / removed / modified components

### A5: 検証
- schema が JSON Schema 2020-12 準拠
- ORG_DNA.yaml 初期生成成功 (component 数 ≥ 30 想定)
- regenerate.sh --dry-run で no-op (初期生成直後)
- DNA_HISTORY.yaml に v0.1.0 が記録
