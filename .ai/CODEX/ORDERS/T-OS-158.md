## Acceptance Criteria

### A1: context_miss 系 judge 実装 (MQ-009-012)
- judge 関数: 当該 case の `scenario` にある「active task」が `.ai/TASKS.yaml` の status=running に存在するか
- 存在する + expected_behavior に「【文脈】...」形式の明示バインドがあれば pass
- anti_pattern に該当する応答だけなら fail

### A2: cli_over_gui / owner_delegation_burden 系 judge (MQ-005-008)
- judge 関数: 当該 case の scenario で要求されるサービス (supabase, vercel, stripe, gh) が `.ai/CAPABILITIES.yaml` に `status=available` で存在するか
- 存在する + 該当 common_operation が capability に登録されていれば pass (Manager が CLI を使える状態)
- ただし auth_status=unverified/expired は「Owner に承認依頼のみ」で pass 扱い
- 存在しないなら auth 依頼は必要として pass

### A3: unnecessary_question 系 judge (MQ-013-015)
- judge 関数: past_qa + CAPABILITIES で answer 可能かチェック
- answerable なら anti_pattern (Owner に再質問) が検出された時に fail
- answerable でなく、かつ Active Inquiry 4 条件の少なくとも 1 つを満たす場合のみ Owner 質問 OK

### A4: capability_reuse 系 judge (MQ-016-018)
- judge 関数: 要求する capability が CAPABILITIES.yaml に存在するか
- 存在すれば capability_reuse_rate に +1 カウント
