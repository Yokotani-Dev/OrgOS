# Codex Work Order — T-OS-156: Context Pack Builder (Pro-P2-6)

> 起動: `bash scripts/codex/run-in-worktree.sh T-OS-156 --keep-worktree`
> **Iron Law: `git add` / `git commit` / `git push` 一切禁止。**

## Background

Owner が外部 LLM (ChatGPT Pro, Codex CLI, Gemini) にプロジェクト state を相談する際、必要 file を redact して一括 export する script。
本 hardening でも 6 round の external review があった (`.ai/REVIEW/T-OS-400/*`)。同様の review を効率化するため Context Pack Builder を作る。

## Required

### `scripts/org/context-pack.sh` (新規)
```bash
# Usage: scripts/org/context-pack.sh [--task T-XXX] [--include-rules] [--include-tasks] [--output PATH]
# Output: /tmp/orgos-context-pack-<ts>.md
# Contents: redacted snapshot of selected files
```

機能:
1. `--include-rules`: `.claude/rules/*.md` を結合
2. `--include-tasks`: `.ai/TASKS.yaml` の active tasks (queued + running) を抽出
3. `--task T-XXX`: 特定 task の context (Work Order + Handoff + artifact manifest)
4. **redaction**: secret pattern (API keys, tokens) を `[REDACTED]` に置換
5. **size limit**: 100KB を超える場合 truncate (large files に対応)

### `scripts/org/redact-secrets.py` (新規 helper)
- regex で API keys / tokens / passwords を検出 → `[REDACTED]`
- aws_access_key, github_token, sk-..., Bearer ..., などの common pattern

### `tests/kernel/test-context-pack.sh` (新規)
- `--include-rules` 実行で rules 結合される
- `--include-tasks` 実行で task 列挙される
- redaction が secret 形式の文字列を置き換える
- size limit が機能する
