## Context

authority-layer.md で設計した OS Mutation Protocol を実行エンジン化する。
「既存 OS ファイルを編集する前に dry-run + diff 確認 + approval チェック」を機械的に実行する script。

## Acceptance Criteria

### A1: scripts/authority/os-mutation-check.sh 実装
呼び出し形式:
```bash
scripts/authority/os-mutation-check.sh \
  --operation edit_manager_md \
  --target .claude/agents/manager.md \
  --diff-file /tmp/proposed-diff.patch
```

出力:
```json
{
  "allowed": true | false,
  "reason": "...",
  "autonomy_level": "execute_with_report" | "ask_before_execute" | "owner_only",
  "approval_required": true | false,
  "next_step": "proceed" | "request_approval" | "reject"
}
```

判定ロジック:
- CONTROL.yaml の allow_os_mutation を読む
- authority-layer.md の os_mutation_protocol.allowed_when_os_mutation_true と照合
- requires_owner_approval / always_forbidden とも照合

### A2: scripts/authority/dry-run.sh
- diff 内容を受け取り、実ファイルを変更せずに検証:
  - YAML/JSON 構文 check
  - 参照整合性 check (dead link にならないか)
  - 最小差分か (大幅変更警告)

### A3: scripts/authority/backup.sh
- 対象ファイルを `.ai/BACKUPS/YYYY-MM-DD/<filename>.bak` に保存
- tar.gz オプションもサポート

### A4: scripts/authority/rollback.sh
- バックアップから指定ファイルを復元
- 復元前の状態も再バックアップ

### A5: scripts/authority/apply.sh (統合エントリポイント)
```bash
scripts/authority/apply.sh \
  --operation edit_claude_md \
  --target CLAUDE.md \
  --patch /tmp/claude.patch
```
内部で:
1. os-mutation-check.sh で allowed 判定
2. allowed=true → backup.sh → dry-run.sh → 実適用
3. allowed=false + approval_required → OWNER_INBOX に記録 (T-OS-173 で実装)
4. allowed=false + reject → 実行拒否

### A6: 読み込む schemas

- .claude/schemas/role-matrix.yaml
- .claude/schemas/approval-workflow.yaml
- .claude/rules/authority-layer.md の判定表

### A7: 検証
```bash
# 新規ファイル作成 → allowed
scripts/authority/os-mutation-check.sh --operation new_file_in_claude_rules --target .claude/rules/test.md
# 既存 CLAUDE.md 編集 → requires_owner_approval
scripts/authority/os-mutation-check.sh --operation edit_claude_md --target CLAUDE.md
# .env 編集 → forbidden
scripts/authority/os-mutation-check.sh --operation edit_env --target .env
```
