## Acceptance Criteria

### A1: check-no-plain-secrets.sh 実装
- pattern-based scanner (gitleaks がない環境でも動く)
- 検出パターン:
  - `sk-[a-zA-Z0-9]{40,}` (OpenAI / Anthropic style)
  - `sk-ant-[a-zA-Z0-9-]{50,}`
  - `ghp_[a-zA-Z0-9]{36}` (GitHub PAT)
  - `gho_[a-zA-Z0-9]{36}` (GitHub OAuth)
  - `xox[baprs]-[a-zA-Z0-9-]{20,}` (Slack)
  - `AKIA[0-9A-Z]{16}` (AWS Access Key)
  - `-----BEGIN [A-Z ]+PRIVATE KEY-----` (PEM)
  - `eyJ[a-zA-Z0-9_-]{20,}\.[a-zA-Z0-9_-]{20,}\.[a-zA-Z0-9_-]{20,}` (JWT)
- `.ai/USER_PROFILE.yaml` 由来 pointer (1password://, op://) は **OK** (pointer は secret 実体ではない)
- gitleaks がインストールされていれば併用 (graceful fallback)
- exit 0 (clean) / exit 1 (secret found) / exit 2 (script error)
- 引数: ファイルパスリスト (空なら staged ファイル)
