# T-OS-112: 非推奨エージェント参照クリーンアップ

> 起動: `bash scripts/codex/run-in-worktree.sh T-OS-112 --keep-worktree`
> Iron Law: git add/commit/push 禁止。

## Background
古い "org-implementer", "org-reviewer" (Codex-based 化前の名残) などへの参照が
`.claude/rules/*.md`, `.claude/commands/*.md`, `.claude/skills/*.md` 等に残存。
新フローは codex-implementer / codex-reviewer + integrator-commit.sh が中心。

## Required
- `.claude/rules/`, `.claude/commands/`, `.claude/skills/`, `.claude/agents/` 配下を
  grep して、deprecated subagent 名 (org-implementer, org-reviewer) を抽出
- 各参照を: (a) codex-implementer / codex-reviewer に置換 OR (b) 文脈に応じて削除
- 過剰な複数 agent 言及があれば 4-5 個 fleet に整理する方向で

## Allowed Paths
- .claude/rules/
- .claude/commands/
- .claude/skills/
- .claude/agents/
- tests/kernel/

## Acceptance
- deprecated reference grep でゼロ 件 (or 明確な意図ある残存)
- 既存 tests pass
- 簡易 test: grep ベースの "deprecated reference count = 0" check
