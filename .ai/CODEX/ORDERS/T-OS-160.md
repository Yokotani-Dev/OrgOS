## Acceptance Criteria

### A1: scripts/memory/check-no-plain-secrets.sh 実装
- USER_PROFILE.yaml / USER_PROFILE.example.yaml / DECISIONS.md 等の .ai/ ファイルを scan
- 検出パターン:
  - `api_key: [非ダミー値]` (プレフィクスが sk-, pk-, xoxb- 等)
  - `password:`, `token:`, `secret:` フィールドの実値
  - AWS/GitHub/Stripe の認証 token 形式
- 検出したら exit 1 + 該当行を出力
- dummy/placeholder 値 (XXX, YYY, <SET_ME>, example.com 等) は無視
- pre-commit から呼び出される想定

### A2: scripts/memory/normalize-lint.sh 実装
- USER_PROFILE.yaml の facts セクションを読む
- 重複 semantic 検出:
  - 類似 id (例: fact_supabase_ref と fact_supabase_project_ref)
  - 同一 scope + 同一 type + 類似 value_ref → warning
- 検出時は warn list を出力 (exit 0)
- 機械的 lint として使える JSON 出力オプション (`--json`)

### A3: scripts/memory/promote-lint.sh 実装
- USER_PROFILE.yaml の facts を scan
- transferability 違反検出:
  - scope: project:x の fact が scope: project:y に複製されている
  - transferability: none の fact が複数 scope に存在
- 違反時は warn (exit 0) + 具体的な fact id を列挙

### A4: .pre-commit-config.yaml の完全化
現在の placeholder を実装に差し替え:
```yaml
repos:
  - repo: https://github.com/gitleaks/gitleaks
    rev: v8.18.0
    hooks:
      - id: gitleaks
  - repo: local
    hooks:
      - id: user-profile-no-secrets
        name: USER_PROFILE に secret 実体が混入していないか検査
        entry: scripts/memory/check-no-plain-secrets.sh
        language: script
        files: '\.ai/.*\.yaml$|\.ai/.*\.md$'
        pass_filenames: false
      - id: memory-normalize-lint
        name: USER_PROFILE の重複 semantic fact 検出
        entry: scripts/memory/normalize-lint.sh
        language: script
        files: '\.ai/USER_PROFILE\.yaml$'
        pass_filenames: false
        stages: [pre-push]
```
