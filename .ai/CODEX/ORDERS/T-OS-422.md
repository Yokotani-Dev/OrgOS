## Background — Dogfood で discovered

T-OS-418 Manager dogfood で `scripts/org/integrator-commit.sh --task-id T-OS-418` が以下で失敗:

```
changed path is outside allowed_paths: .ai/queue/integration/processing/
failed queue item: .ai/queue/integration/failed/T-OS-418.20260515T184114Z.json
```

### Root cause

integrator-commit.sh は queue item を pending/ → processing/ に **先に** 移動し、その後で changed_files を `git status --porcelain` で収集する。結果として:

```
git status --porcelain (after queue move):
 D .ai/queue/integration/pending/T-OS-418.json
?? .ai/queue/integration/processing/T-OS-418.json
?? docs/kernel-v2/dogfood.md         ← actual user diff
```

allowed_paths check は **3 paths 全て** をチェックする。lease.allowed_paths = `docs/kernel-v2/` のため、queue paths が「outside allowed_paths」として弾かれる。

### KRT-007 が見落とした理由

KRT-007 test fixture では integrator-commit.sh を temp repo で実行し、queue 移動の前後で git status を観察していない (or 観察しても allowed_paths が temp repo の queue paths をカバーしている)。real-world では:
- queue は同じ worktree 内
- user diff の allowed_paths は queue paths を含まない (普通そうしない)

## Fix

`scripts/org/integrator-commit.sh` の changed_files 収集 Python script (L270-282) で `.ai/queue/integration/**` を filter out する。

```python
# 旧
output = subprocess.check_output(["git", "-C", worktree, "status", "--porcelain"], text=True)
paths = []
for line in output.splitlines():
    if len(line) < 4:
        continue
    path = line[3:]
    if " -> " in path:
        path = path.split(" -> ", 1)[1]
    paths.append(path)
for path in sorted(set(paths)):
    print(path)

# 新
output = subprocess.check_output(["git", "-C", worktree, "status", "--porcelain"], text=True)
INTERNAL_PATHS = (
    ".ai/queue/integration/",  # integrator-managed queue state transitions
    ".ai/leases/",             # lease registry transitions
    ".ai/artifacts/",          # artifact store (managed by collect-artifacts)
)
paths = []
for line in output.splitlines():
    if len(line) < 4:
        continue
    path = line[3:]
    if " -> " in path:
        path = path.split(" -> ", 1)[1]
    if any(path.startswith(p) for p in INTERNAL_PATHS):
        continue  # skip integrator-managed paths
    paths.append(path)
for path in sorted(set(paths)):
    print(path)
```

### git add も同様に filter する必要

L368 の `git -C "$worktree_path" add -- "${changed_files[@]}"` も changed_files から filter された結果を使うので OK (changed_files に integrator paths が入らない)。

ただし元 git status は internal paths も見せるが add しない、という挙動は OK。
