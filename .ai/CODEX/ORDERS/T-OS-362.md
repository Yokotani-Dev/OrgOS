## Context

Owner Issue (2026-05-10): 複数 Codex CLI が同じ working directory で起動すると git checkout が裏で発生し、Manager セッションが意図しないブランチに切り替わる事故が発生。

OrgOS の `.claude/settings.json` は `additionalDirectories: [".worktrees"]` を宣言しており、worktree 並列を **設計レベルでサポート**しているが、実運用で活用されていなかった。

本タスクで Codex 起動 wrapper を作成し、各 Codex を別 worktree に隔離する。

## Acceptance Criteria

### A1: run-in-worktree.sh
```bash
bash scripts/codex/run-in-worktree.sh <TASK_ID> [--keep-worktree]
```
- 動作:
  1. `.worktrees/<TASK_ID>` に worktree を作成 (`git worktree add .worktrees/<TASK_ID> HEAD`)
  2. その worktree 内で `/opt/homebrew/bin/codex exec --full-auto --skip-git-repo-check --output-last-message ../../.ai/CODEX/RESULTS/<TASK_ID>.txt - < ../../.ai/CODEX/ORDERS/<TASK_ID>.md`
  3. 完了後 `git worktree remove .worktrees/<TASK_ID>` (--keep-worktree がなければ)
- worktree 作成失敗時: エラーメッセージ + exit 1
- TASK_ID 未指定: usage 表示 + exit 2

### A2: cleanup-worktrees.sh
- 引数なし: 24h 以上前に作成された `.worktrees/T-OS-*` を全削除
- `--all`: 全 `.worktrees/` を削除 (確認 prompt 付き)
- `--dry-run`: 削除対象のみ表示

### A3: .gitignore
- `.worktrees/` を追加 (既存設定を破壊しない、append のみ)

### A4: README
- 旧コマンド: `/opt/homebrew/bin/codex exec --full-auto - < .ai/CODEX/ORDERS/T-XXX.md`
- 新コマンド: `bash scripts/codex/run-in-worktree.sh T-XXX`
- 利点 (なぜ worktree 化するか)
- 移行手順 (Manager 側更新は別タスク T-OS-362b)

### A5: 検証
- 新規 worktree で codex exec が動作 (実行成功)
- 並列 2 worktree で同時実行可能 (各々別ブランチで動く)
- cleanup スクリプトで worktree 削除成功
- shellcheck 通過
