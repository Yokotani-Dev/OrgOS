## Acceptance Criteria

### A1: check-branch-consistency.sh に --block 追加
- 既存の --report mode 維持
- 新規 `--block-if-mismatch`: mismatch なら exit 非 0 + 明確なエラーメッセージ
- 新規 `--require-lock`: `.claude/state/git.lock` を保持していなければ exit 非 0

### A2: pretool_policy.py 強化 (additive)
- T-OS-371 で kernel block + T-OS-361 で branch check + T-OS-363 で git lock が既に組み込まれている
- 本タスクで:
  - git mutation tool (commit / checkout / merge / rebase / stash / push) 検出時に **必ず** acquire-lock を実行
  - lock 取得失敗 → block
  - branch mismatch (--block-if-mismatch 経由) で block
- 既存機能を破壊しない (additive)

### A3: run-in-worktree.sh に T-OS-372 integrate
- worktree 作成前: `bash scripts/codex/pre-exec-validate.sh <TASK_ID>` を call、failure なら abort
- codex exec 完了後: `bash scripts/codex/post-exec-audit.sh <TASK_ID> <worktree_path>` を call、failure なら revert
- pre-exec / post-exec の出力を log に記録

### A4: 検証
- check-branch-consistency.sh --block-if-mismatch で mismatch 時に exit 非 0
- pretool_policy.py が git commit 前に lock 必須化
- run-in-worktree.sh が pre/post を call (test fixture)
- 既存テストを破壊しない
- shellcheck 通過
