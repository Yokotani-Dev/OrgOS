## Context

T-OS-361 でブランチ整合性チェックが入ったが、複数セッションが同時に git 操作する場合の **mutual exclusion** はまだない。
本タスクで flock ベースの協調ロックを導入する。

## Acceptance Criteria

### A1: acquire-lock.sh
- `bash scripts/git/acquire-lock.sh [--timeout 30]`
- flock で `.claude/state/git.lock` を排他取得
- timeout 内に取得できなければ exit 1 + エラーメッセージ (現在の lock holder pid 表示)
- 取得成功なら lock holder info (pid, sessionId, acquired_at) を lock file に記録

### A2: release-lock.sh
- `bash scripts/git/release-lock.sh`
- flock を release (script exit で自動 release されるが、明示的な release も提供)

### A3: pretool_policy.py 統合
- T-OS-361 で追加された branch consistency check の前段に lock acquire を追加
- destructive git 操作 (commit/checkout/merge/rebase/stash/push) のみ lock 必須
- read-only 操作 (status/log/diff) は lock 不要
- T-OS-361 の機能を破壊しない (additive のみ)

### A4: 検証
- 2 並列で acquire-lock.sh を実行 → 1 つ目が取得、2 つ目が timeout
- 1 つ目が release 後、2 つ目が取得成功
- pretool_policy.py の既存テスト (あれば) を破壊しない
- shellcheck と python lint を通す
