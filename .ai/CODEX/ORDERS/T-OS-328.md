## Context

現状 `.ai/INTELLIGENCE/` は空 (.gitkeep のみ)。Phase 2 SYNTHESIS §3 で Intelligence 層を稼働させる。

機能:
1. **collect.sh**: config.yaml に列挙された外部 source (Anthropic blog, OpenAI changelog, Vercel blog 等) から weekly fetch、raw/ に保存
2. **summarize.sh**: raw/ を読み weekly summary を生成 (fixture LLM stub OK)、weekly/ に保存
3. **emit-oip.sh**: summary から OIP (OrgOS Improvement Proposal) を生成、reports/ に保存、必要なら proposal として T-OS-324 synthesizer に渡す

## Acceptance Criteria

### A1: config.yaml
```yaml
sources:
  - id: anthropic-blog
    type: rss
    url: https://www.anthropic.com/news/rss
    fetch_cadence: weekly
    cache_ttl: 24h
  - id: openai-changelog
    ...
```
- 5+ source を初期登録 (Anthropic / OpenAI / Vercel / Supabase / Claude Code release notes 等)

### A2: collect.sh
- config.yaml を読み、各 source を fetch
- network failure は log_only で skip (graceful)
- raw/<source_id>/<YYYY-MM-DD>.json or .xml に保存

### A3: summarize.sh
- raw/ の最新 7 日分を読み、weekly/<YYYY-WW>.md を生成
- LLM 呼び出しは fixture stub OK (本タスクでは「pipeline が動く」が目的)

### A4: emit-oip.sh
- weekly/ の最新 summary を読み、OIP candidate を reports/ に出力
- フォーマット: OIP-XXX (id), title, source_refs, suggested_action

### A5: 検証
- 全 4 script + config + 3 ディレクトリ
- collect.sh の network failure graceful (offline でもエラーにならない)
- summarize / emit-oip は fixture で動作
