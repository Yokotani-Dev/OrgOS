## Acceptance Criteria

### A1: `.ai/CONTROL.yaml` の `codex.approval` コメントを更新
- `untrusted / on-failure / on-request / never` の列挙を残しつつ、`on-failure` に **「DEPRECATED (Codex 0.102+)」** を注記
- 推奨値: 対話=`on-request`、非対話/CI=`never`

### A2: `codex exec` に `-a` が無いことを明記
- `CODEX_WORKER_GUIDE.md` に専用セクション「承認モードと sandbox」を新設
- `codex exec` では `-a/--ask-for-approval` は**使えない**（対話 `codex` と `codex resume` でのみ使える）
- 非対話で制御する場合:
  - 承認不要: `--full-auto`（= `-a on-request --sandbox workspace-write` の省略形）
  - 完全無承認: `-c approval_policy=never` または `--dangerously-bypass-approvals-and-sandbox`（最終手段）
- `.ai/CONTROL.yaml` の `codex.approval` コメントにもこの注意書きを1行で入れる

### A3: 結果の構造化取得パターン
- `CODEX_WORKER_GUIDE.md` に「結果の取り出し」セクション追加:
  - `--output-last-message <FILE>`: 最終メッセージをファイルに書き出す（`.ai/CODEX/RESULTS/<TASK_ID>.txt` 等）
  - `--output-schema <FILE>`: 最終応答の JSON Schema を指定して構造化
  - `--json`: 全イベントを JSONL で stdout に流す
  - `.ai/CODEX/RESULTS/<TASK_ID>.md` の生成パイプラインでこれらを使う例を記載

### A4: セッション再開（resume）の手順
- `CODEX_WORKER_GUIDE.md` に「セッション再開」セクション追加:
  - `codex exec resume <SESSION_ID>` / `codex exec resume --last`
  - 長尺タスクを Tick 跨ぎで再開する運用指針
  - `CODEX_THREAD_ID` 環境変数がセッション識別に使える旨

### A5: stdin piping サポート
- `CODEX_WORKER_GUIDE.md` に「Work Order の投入方法」サブセクション追加:
  - 方法1: ファイル名引数（既存）
  - 方法2: stdin piping（0.118+）
    - `cat .ai/CODEX/ORDERS/T-123.md | codex exec --full-auto -`
    - または heredoc
- `.claude/agents/manager.md` の Codex 起動例にも1行追加（長いプロンプトは stdin 推奨）

### A6: model / profile / 階層 config の方針
- `.ai/CONTROL.yaml` の `codex:` セクションを拡張:
  ```yaml
  codex:
    auto_exec: true
    sandbox: "workspace-write"
    approval: "on-request"
    # モデル指定は行わない（config.toml のデフォルトを使用）
    # model: "gpt-5.4-codex"  # 通常は未指定のまま
    profile: ""                # 必要時のみ `-p` で渡すプロファイル名
    # 階層 config: /etc/codex/config.toml → ~/.codex/config.toml → .codex/config.toml
    # プロジェクト固有設定は `.codex/config.toml` に置く
  ```
- `CODEX_WORKER_GUIDE.md` にも「モデル・プロファイル・階層 config」サブセクション追加
  - **Iron Law: Manager/Worker は `-m` を付けてモデル指定しない。** config.toml 側で管理する
  - `~/.codex/config.toml` とリポジトリ直下 `.codex/config.toml` のマージ順序を明記
