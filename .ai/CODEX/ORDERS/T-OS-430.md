## Background

Owner pain: 「並列で開発している時に別のブランチに迷い込む現象が多発」

このパターンは 2026-05-10 incident と同型。kernel v2 は OrgOS-Dev でのみ enforce、他 10+ project repo は旧 OrgOS のまま (Codex 自由 commit / Manager raw checkout)。

## 目的

`scripts/org/deploy-kernel-v2.sh` を作成し、Owner が `bash scripts/org/deploy-kernel-v2.sh /path/to/other-repo` で **必要最小 kernel v2 ファイル** を配布できるようにする。

## 配布モデル

GPT 4th Q22 推奨の **hybrid**:
- kernel code を `~/.orgos/kernel/v0.2.0/` に置く (global)
- 各 repo の `.claude/hooks/`, `scripts/codex/`, `scripts/org/` は symlink (or copy with --copy flag)
- 各 repo の `.ai/` は project-local
- 各 repo に `.orgos-kernel-version` ファイルで pin

ただし、初版は **simple copy** で十分 (Owner pain を即対応)。symlink は subsequent improvement。

## CLI 仕様

```bash
# Owner usage:
bash scripts/org/deploy-kernel-v2.sh /path/to/target-repo [--mode warn|enforce-protected-only|enforce-all] [--dry-run] [--force]

# Defaults:
--mode enforce-protected-only  # 推奨: IntegratorOnlyCommit + ProtectedBranchNoTouch のみ enforce、他は warn
                                # (Owner pain を即対応、他 invariant は段階的)
                                
--dry-run                      # 実行せず計画表示
--force                        # 既存 file を上書き (非 --force なら conflict で abort)
```

### enforce-protected-only mode (推奨 default)

```json
{
  "schema_version": "orgos.kernel-mode.v2",
  "default": "warn",
  "invariants": {
    "IntegratorOnlyCommit": "enforce",
    "PerTaskWorktree": "warn",
    "ProtectedBranchNoTouch": "enforce",
    "LeaseBeforeWrite": "warn",
    "StateMutationViaOrgTool": "warn",
    "DurableArtifactBeforeCleanup": "warn",
    "OwnerApprovalForIrreversibleOps": "disabled",
    "DangerousShell": "warn",
    "KernelSelfModification": "warn",
    "IntegratorIsScriptNotAgent": "warn"
  }
}
```

これだけで Owner pain (branch confusion + raw commit) は止まる。

### 動作

1. `target-repo` が git repo であることを確認
2. `.claude/`, `scripts/codex/`, `scripts/org/`, `.claude/schemas/`, `.claude/evals/`, `.ai/queue/`, `.ai/` を作成
3. 上記 21 ファイルを **本 repo (OrgOS-Dev)** から target-repo へコピー
4. `target-repo/.orgos-kernel-version` に `v0.2.0` を書き込み
5. `target-repo/.claude/state/kernel-mode.json` を指定モードで作成
6. Summary を stdout: deployed files / skipped / target version / mode

### Allowed Paths

```
scripts/org/deploy-kernel-v2.sh
tests/kernel/test-deploy-kernel-v2.sh
```
