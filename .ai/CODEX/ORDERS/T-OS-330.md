# Codex Work Order — T-OS-330: TASKS.yaml archive automation

> 起動: `bash scripts/codex/run-in-worktree.sh T-OS-330 --keep-worktree`
> **Iron Law: `git add` / `git commit` / `git push` 一切禁止。**

## Background
`.ai/TASKS.yaml` の done/cancelled/superseded task が肥大化 (現状 67+ tasks)。
GPT-5.5 5th round で `.ai/TASKS_ARCHIVE.yaml` への分離が提案された。

## Required Implementation

### scripts/org/archive-tasks.py (新規)
- `.ai/TASKS.yaml` から status=done/cancelled/superseded の task を抽出
- `.ai/TASKS_ARCHIVE.yaml` に移動 (タイムスタンプ付き)
- `.ai/TASKS.yaml` から削除 (atomic write)
- validate-tasks-yaml.py で書き出し前後検証

### CLI
```bash
scripts/org/archive-tasks.py [--dry-run] [--threshold-days 30]
```
- `--dry-run`: 移動対象表示のみ
- `--threshold-days N`: N 日以上経過した task のみ移動 (default: 全て)

### Tests
`tests/kernel/test-archive-tasks.sh`:
- dry-run で移動対象正しく表示
- done task が ARCHIVE.yaml に移動、TASKS.yaml から消える
- 移動後 validate-tasks-yaml.py 両方 pass

### Allowed paths
- scripts/org/archive-tasks.py
- tests/kernel/test-archive-tasks.sh
- docs/kernel-v2/archive-runbook.md (任意 runbook)
