## Acceptance Criteria

### A1: archive-done.sh
- TASKS.yaml を読み、`status: done`、`status: archived`、`status: superseded` の task を抽出

- コメント (`# === ... ===`) は **保持**
- セクションヘッダ間の空行も保持
- `--dry-run` で削除予定 task ID を出力のみ

### A2: 初回実行
- 現状の done/archived/superseded を全て TASKS_ARCHIVE.yaml に移動
- 移動件数を report

### A3: Runbook (.ai/RUNBOOKS/archive-tasks.md)

- どう実行するか (`bash scripts/tasks/archive-done.sh`)
- 失敗時の rollback (TASKS.yaml backup + restore)

### A4: 検証
- 冪等性: 2 回実行して結果が同じ
- セクションヘッダ保持
- 初回実行で M-PHASE-6 関連の done task が ARCHIVE へ移動
- Runbook が Owner / Manager 双方に分かりやすい
