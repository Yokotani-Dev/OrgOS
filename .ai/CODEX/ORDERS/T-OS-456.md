# Codex Work Order — T-OS-456: Invariant #7 PlanContractRequired

## Background

Add 8th constitutional invariant: PlanContractRequired. Codex/Manager Edit ops on a task with no `.ai/plans/<task_id>.plan.yaml` must be denied. This forces upfront planning before any work.

Current 7 invariants in `.claude/hooks/policy_core.py`:
1. IntegratorOnlyCommit
2. PerTaskWorktree
3. ProtectedBranchNoTouch
4. LeaseBeforeWrite
5. StateMutationViaOrgTool
6. DurableArtifactBeforeCleanup
7. OwnerApprovalForIrreversibleOps

Add #8: PlanContractRequired

## Required

### `.claude/hooks/policy_core.py` (拡張)

Add new invariant:
- Resolve task_id from current lease (`.ai/leases/active/*.json`)
- Check `.ai/plans/<task_id>.plan.yaml` exists
- If missing, deny Edit/Write/Bash{git commit} with reason "PlanContractRequired: .ai/plans/<task_id>.plan.yaml not found"
- Skip for: read-only ops, manager bootstrap context, Edit on .ai/plans/ itself

Mode flag in `.claude/state/kernel-mode.json`:
- Default: `warn` (T-OS-456 introduces it; T-OS-458 flips to `enforce`)

### `tests/kernel/test-plan-contract-required.sh` (新規)

- Edit without `.plan.yaml` → denied (in enforce mode)
- Edit with valid `.plan.yaml` → allowed
- Edit on `.ai/plans/` itself → allowed (bootstrap exception)
- warn mode → allowed but warned
- disabled mode → allowed silently
- Lease info correctly resolves task_id
