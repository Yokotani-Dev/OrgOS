# Codex Work Order — T-OS-457: integrator verifies .plan.yaml

> 起動: `bash scripts/codex/run-in-worktree.sh T-OS-457 --keep-worktree`
> **Iron Law: `git add` / `git commit` / `git push` 一切禁止。**

## Background

Before integrator-commit.sh commits, it must verify:
1. `.ai/plans/<task_id>.plan.yaml` exists
2. Plan passes schema validation
3. Files changed in worktree are within `allowed_paths` listed in plan

This is a sanity gate at the integration boundary, complementing Invariant #8 (T-OS-456) which enforces at write time.

## Required

### `scripts/org/integrator-commit.sh` (拡張)

Add new step BEFORE git commit:
1. Read `task_id` from queue item
2. Validate `.ai/plans/<task_id>.plan.yaml` against `.claude/schemas/plan-contract.v1.json`
3. Collect changed files via `git diff --name-only HEAD`
4. Verify each changed file is covered by some entry in plan's `allowed_paths`
5. On failure: move queue item to `failed/`, emit IntegrationFailed event, exit non-zero

### `tests/kernel/test-integrator-plan-contract.sh` (新規)

- integrator commits when plan.yaml valid and changes match allowed_paths
- integrator rejects commit when plan.yaml missing
- integrator rejects commit when changed file outside allowed_paths
- integrator rejects commit when plan.yaml invalid schema

## Allowed Paths

- `scripts/org/integrator-commit.sh`

- `tests/kernel/run-kernel-tests.sh`
