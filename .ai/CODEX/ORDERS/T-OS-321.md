## Context

現状の `.ai/OWNER_INBOX.md` は「質問羅列」形式で、Owner が読んで考えて回答する負荷が高い。さらにテスト残滓 (T-TEST-NOWAIT 等) が **12 日以上 pending で放置** されており、Owner が定期的にクリーンアップすべきものを Owner 自身が知らないで放置している。

Phase 2 SYNTHESIS の §3.3 で定義した **Decision Table format** に転換し、各 entry に `recommendation` / `risk` / `default_if_no_response` / `deadline` を必須化する。未応答 7 日 (configurable) で `auto_apply` (synthetic Owner approve のみ)。

- `.ai/REVIEW/T-OS-300/SYNTHESIS.md` §3.3 + §5

## Acceptance Criteria

### A1: Decision Card Schema
`.claude/schemas/decision-card.yaml` を新規作成:

```yaml
$schema: orgos/decision-card/v1
fields:
  id:
    pattern: "^D-\\d{4}-\\d{2}-\\d{2}-\\d{3}$"
    required: true
  type:
    enum: [type_a_direction, type_b_preference, type_c_authentication, type_d_notification]
    required: true
  decision:
    type: string
    description: "1 行で何を決めるか"
    required: true
  recommendation:
    enum: [APPROVE, DEFER, REJECT]
    required: true
  recommendation_reason:
    type: string
    required: true
  risk:
    enum: [low, medium, high, critical]
    required: true
  options:
    type: array
    items:
      - key: string  # A / B / C
        label: string
        consequence: string
        is_recommended: bool
    required: true
  default_if_no_response:
    enum: [auto_apply, defer_7d, escalate, no_op]
    required: true
  deadline:
    type: ISO8601
    required: true
  synthetic_owner_judgment:
    type: object
    optional: true
    fields:
      verdict: enum [approve, reject, needs_more_context]
      confidence: 0.0..1.0
  status:
    enum: [pending, approved, rejected, auto_applied, expired]
    default: pending
  resolved_at: ISO8601
```

### A2: Template
`.ai/TEMPLATES/OWNER_INBOX.md` を新規作成 — Decision Console 形式の skeleton:

```markdown
# OWNER INBOX (Decision Console)

> Manager から Owner への決済依頼。各 Card に推奨 + デフォルト動作付き。
> 未応答 7 日で `default_if_no_response` が自動適用される (synthetic Owner approve のみ)。
> 質問への回答は: `echo "D-XXX A" >> .ai/OWNER_COMMENTS.md` または「D-XXX A」と発話。

## 高優先度決済 (response < 24h 推奨)
(なし)

## 中優先度決済 (response < 7d)
(なし)

## 低優先度決済 (response < 30d)
(なし)

## Archived (resolved or expired)
(なし)
```

### A3: 既存 OWNER_INBOX.md の format 転換
現状の `.ai/OWNER_INBOX.md` を Decision Table format に転換:

1. 既存の各 pending entry を Decision Card に変換
   - `id` を `D-2026-05-01-XXX` 形式で発番
   - 既存 entry が「質問形式」なら Manager 視点で `decision` を再記述
   - `recommendation` は最も保守的な選択を提示
   - `risk` は Owner 影響範囲から推定
   - `default_if_no_response` は status: stale なら `expired`、新規なら `defer_7d`

2. **テスト残滓を archive**
   - T-TEST / T-TEST-NOWAIT / T-TEST-EXPIRE 等の test entry を `## Archived` セクションへ移動
   - status: expired

3. **OrgOS-dev 用と一般プロジェクト用を分離**
   - 既存 entry が OrgOS 自体の改修関係なら 高優先度
   - 一般プロジェクトの kickoff 質問なら 中優先度

### A4: Helper Scripts
`scripts/inbox/` 配下に:
- `add-decision.sh` — Decision Card を新規追加 (引数で決定内容、自動 ID 発番)

- `archive.sh` — resolved / expired を Archived セクションへ移動
