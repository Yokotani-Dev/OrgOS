#### Part 1: Autonomy Level 4 段階
```yaml
autonomy_levels:
  - id: silent_execute
    description: Manager 完全自律実行、Owner 通知不要
    applicable_to:
      - reversible operations
      - low cost
      - security: none
      - scope: local
    examples: [grep, format, lint]
  
  - id: execute_with_report
    description: Manager 自律実行、完了時に簡潔報告
    applicable_to:
      - reversible operations
      - cost: low
      - security: low
    examples: [新規ファイル作成, 既存 scripts/memory/ 編集]
  
  - id: ask_before_execute
    description: Owner に推奨付きで問い合わせ、承認後実行
    applicable_to:
      - irreversible operations
      - security: medium+
      - cost: medium+ (課金)
    examples: [DB migration, 外部 API 呼び出し, 本番相当環境への push]
  
  - id: owner_only
    description: Owner のみが実行可能 (Manager は提案・準備のみ)
    applicable_to:
      - destructiveness: external
      - security: high
      - legal/billing/compliance
    examples: [本番 deploy, データ削除, 契約承認]
```

#### Part 2: OS Mutation Protocol (ISSUE-OS-001 解決)
```yaml
os_mutation_protocol:
  # CONTROL.yaml の allow_os_mutation=true が前提
  # 以下の操作を新たに許容
  
  allowed_when_os_mutation_true:
    - new_file_in_claude_rules        # .claude/rules/*.md 新規
    - new_file_in_claude_schemas      # .claude/schemas/*.yaml 新規
    - new_script_in_scripts           # scripts/*/ 新規
    - edit_existing_claude_rules_new_since_epoch  # 本セッション以降に作成されたファイルは編集可
    - update_gitignore                # .gitignore への追加のみ
  
  requires_owner_approval:
    - edit_claude_md                  # CLAUDE.md 編集
    - edit_manager_md                 # .claude/agents/manager.md 編集
    - edit_agents_md                  # AGENTS.md 編集
    - edit_control_yaml               # .ai/CONTROL.yaml の非 runtime フィールド
    - delete_existing_rule            # 既存ルール削除
  
  always_forbidden:
    - edit_kernel_files               # .claude/evals/KERNEL_FILES 記載ファイル
    - bypass_eval                     # eval skip
    - disable_rationalization_prevention
  
  # 実行前の手順
  preflight:
    - dry_run: "diff 内容を確認"
    - approval: "ask_before_execute level 以上ならレビュータスク生成"
    - backup: ".ai/BACKUPS/ に変更前を保存"
  
  # 実行後
  postflight:
    - verify: "変更内容を eval で検証"
    - record: "DECISIONS.md に OS-MUTATION-XXX として記録"
    - rollback_ready: "git commit で rollback 可能な状態"
```
