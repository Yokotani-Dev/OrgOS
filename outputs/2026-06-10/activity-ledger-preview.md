## 2. 期間指定もできる

| 打つもの | 出るもの |
|---------|---------|
| `/org-journal` | 今日の一覧（上の形式） |
| `/org-journal 2026-06-08` | 指定した日の一覧 |
| `/org-journal --days 7` | 今週やったことのまとめ |

## 3. 自然言語でも聞ける（MCP 連携 — どのリポジトリからでも）

**あなた**:「先週、TMC で決めたことって何だっけ？」

**Claude**:「先週 TMC では 3 件の意思決定が記録されています：
　① 6/4 招待レス機構の Phase2 分割
　② 6/5 Webhook 検証を先行
　③ 6/6 トークン TTL を 24h に短縮」

**あなた**:「メモして: 来週は V-Platform の請求まわり優先」

**Claude**:「記録しました。ジャーナルの『考えたこと』に表示されます」

---

## 4. 記録はほぼ全自動（あなたの操作は不要）

| 何が記録される | きっかけ | あなたの操作 |
|--------------|---------|------------|
| セッション開始/終了 | Claude Code を開く/閉じる | なし（自動） |
| タスク完了・コミット | OrgOS の既存イベントから自動取込 | なし（自動） |
| Manager の重要判断 | Manager が判断したとき | なし（自動） |
| あなたのメモ・思いつき | 「メモして」と言ったとき | 一言いうだけ |

---

## 5. データの保存場所

```
~/.orgos/activity/          ← ホームフォルダ配下（リポジトリの外）
  events-202606.jsonl       ← 6月分のログ（1行=1イベントのテキスト）
  events-202607.jsonl       ← 7月分…
```

- どのリポジトリで作業しても**ここ 1 箇所**に集まる
- git には入らない（プライベート）
- 将来の Web ダッシュボードもこのデータをそのまま読める
